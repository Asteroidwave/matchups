# Port Configuration Guide ## Correct Port Setup - **Frontend (Next.js)**: Port **3000** (default) - **Backend (Express)**: Port **3001** ## Environment Variables Your `.env.local` file should have: ```env NEXT_PUBLIC_BACKEND_URL=http://localhost:3001 ``` ## Why This Matters The frontend code uses `process.env.NEXT_PUBLIC_BACKEND_URL || 'http://localhost:3001'` to connect to the backend. If the frontend is on a different port, the API calls will still work, but it's confusing. ## How to Fix Port Conflicts ### If Port 3000 is in use: 1. **Find what's using it**: ```bash lsof -ti:3000 ``` 2. **Kill the process**: ```bash kill -9 $(lsof -ti:3000) ``` 3. **Restart frontend**: ```bash npm run dev ``` ### If Backend Port 3001 is in use: 1. **Check backend/.env** - make sure it's set to 3001 2. **Or change backend port** in `backend/src/index.ts`: ```typescript const PORT = process.env.PORT || 3001; ``` ## Quick Fix If you see "Port 3000 is in use, trying 3001 instead": 1. **Kill processes on port 3000**: ```bash kill -9 $(lsof -ti:3000) ``` 2. **Kill any Next.js processes**: ```bash pkill -f "next dev" ``` 3. **Restart frontend**: ```bash npm run dev ``` 4. **Verify**: - Frontend should be on `http://localhost:3000` - Backend should be on `http://localhost:3001` ## Testing After fixing, verify: - Frontend: `http://localhost:3000` loads - Backend: `http://localhost:3001/api/health` (if you have a health endpoint) - API calls from frontend work correctly
