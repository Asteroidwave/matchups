# What Happens After Stage 2 (Preview & Create Contest) ## Overview After you successfully fetch track data (Stage 1) and preview it (Stage 2), here's the complete flow: ## Step-by-Step Flow ### 1. **Create Contest (Stage 2)** - Click "Create Contest" button after viewing the preview - Contest is created in Supabase with: - `status: 'draft'` - Contest exists but matchups aren't calculated yet - `is_active: false` - Contest is NOT visible in lobby yet - `matchup_types: []` - No matchup types selected yet - `track_data_id` - Links to the fetched track data ### 2. **Manage Contests Section** - The new contest appears in the "Manage Contests" table - Status shows as "Draft" - Matchups count shows "0" - **The visibility toggle is DISABLED** (grayed out) because status is not 'ready' ### 3. **Set Matchup Types** - Click the "️ Set Matchups" button (gear icon) next to the contest - A dialog opens where you can: - Select matchup types (Jockey vs Jockey, Trainer vs Trainer, Sire vs Sire, Mixed) - Configure calculation settings for each matchup type: - Count (how many matchups to generate) - Tolerance (point difference threshold) - Prefer 1v1 (preference for head-to-head matchups) - Max Attempts (retry limit for generation) - See a summary and estimated calculation time - Click "Calculate Matchups" to start the process ### 4. **Matchup Calculation (Background Process)** - Status changes to "Processing" - A background service calculates matchups based on: - The fetched track data - Your selected matchup types - Your calculation settings - Progress is tracked in the `operations` table - This process can take several minutes depending on data size and settings ### 5. **Matchups Ready** - Once calculation completes: - Status changes to "Ready" - Matchups count updates to show the actual number - Matchups are stored in the `matchups` table - **The visibility toggle becomes ENABLED** (clickable) ### 6. **Toggle Visibility** - Click the visibility toggle switch to make the contest visible - `is_active` changes to `true` - Contest now appears in the lobby for all users - Users can click "Enter" to join the contest ## Status Flow Diagram ``` Stage 1: Fetch Data Stage 2: Preview & Create Status: 'draft' (Visibility Toggle DISABLED) Set Matchup Types Click "Calculate Matchups" Status: 'processing' (Matchups being calculated) Status: 'ready' (Visibility Toggle ENABLED) Toggle Visibility ON Contest appears in Lobby Users can join ``` ## Important Notes 1. **Visibility Toggle is Disabled Until Ready**: You cannot make a contest visible until matchups are calculated. This ensures users only see contests with actual matchups available. 2. **Calculation Time**: Matchup calculation can take 1-10+ minutes depending on: - Number of horses/jockeys/trainers/sires - Number of matchup types selected - Calculation settings (tolerance, count, etc.) - Server load 3. **Processing Status**: While calculating, the status badge shows "Processing" with a pulsing animation. The page will automatically refresh to show updates. 4. **Error Handling**: If calculation fails, status may revert to 'draft' and you'll need to retry setting matchup types. 5. **Multiple Matchup Types**: You can select multiple types (e.g., JvJ + TvT) and they'll all be calculated together. ## Troubleshooting - **"Request timeout" errors**: Increased timeout to 30 seconds for Supabase free tier. If still timing out, check your internet connection or Supabase status. - **Buttons become inactive after switching tabs**: Session refresh is now handled automatically when you return to the tab. If issues persist, refresh the page. - **Contest not appearing in lobby**: Check that: 1. Status is 'ready' 2. Visibility toggle is ON (green) 3. You're logged in (contests are public but entry requires auth)
