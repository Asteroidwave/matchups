# RLS Policies for Admin Features ## Problem Admins need to: 1. Read all user profiles (not just their own) 2. Update any user's bankroll (not just their own) 3. Read/manage all contests ## Solution Run these SQL commands in Supabase SQL Editor: ### 1. Allow Admins to Read All Profiles ```sql -- Drop existing policy if it exists DROP POLICY IF EXISTS "Admins can view all profiles" ON profiles; -- Create policy: Admins can view all profiles CREATE POLICY "Admins can view all profiles" ON profiles FOR SELECT USING ( auth.uid() = id -- Users can view their own OR EXISTS ( SELECT 1 FROM profiles WHERE id = auth.uid() AND is_admin = true ) -- OR user is an admin ); ``` ### 2. Allow Admins to Update Any User's Bankroll ```sql -- Drop existing policy if it exists DROP POLICY IF EXISTS "Admins can update any profile" ON profiles; -- Create policy: Admins can update any profile CREATE POLICY "Admins can update any profile" ON profiles FOR UPDATE USING ( auth.uid() = id -- Users can update their own OR EXISTS ( SELECT 1 FROM profiles WHERE id = auth.uid() AND is_admin = true ) -- OR user is an admin ); ``` ### 3. Allow Admins to Manage Contests ```sql -- Drop existing policies if they exist DROP POLICY IF EXISTS "Admins can manage contests" ON contests; DROP POLICY IF EXISTS "Everyone can view active contests" ON contests; -- Admins can do everything with contests CREATE POLICY "Admins can manage contests" ON contests FOR ALL USING ( EXISTS ( SELECT 1 FROM profiles WHERE id = auth.uid() AND is_admin = true ) ); -- Everyone can view active contests CREATE POLICY "Everyone can view active contests" ON contests FOR SELECT USING (is_active = true); ``` ## Test After running these: 1. **Test Admin Reading Users**: - Login as admin - Go to Admin Users - Should see all users 2. **Test Admin Updating Bankroll**: - Login as admin - Go to Admin Users - Adjust a user's bankroll - Should update successfully 3. **Test Contest Management**: - Login as admin - Go to Admin Contests - Should see all contests - Should be able to create/delete/update ## Notes - These policies use the `is_user_admin()` function we created earlier - Admins can read/update all profiles - Regular users can only read/update their own profile - Everyone can view active contests - Only admins can create/update/delete contests
