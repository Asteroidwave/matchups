# Answers to Your Questions ## 1. How to Delete a User Completely **Problem:** You can't delete from Authentication Users tab in Supabase. **Solution:** Use SQL to delete from both tables. ### Steps: 1. **Go to Supabase Dashboard SQL Editor** 2. **Run this SQL** (replace `ritho@ralls.com` with the email): ```sql -- Find the user ID first SELECT id, email FROM auth.users WHERE email = 'ritho@ralls.com'; -- Copy the UUID, then run: DELETE FROM profiles WHERE id = 'USER-UUID-HERE'; DELETE FROM auth.users WHERE id = 'USER-UUID-HERE'; ``` **Or use the complete script in `docs/DELETE_USER.sql`** - it has both methods. **Why this works:** - Supabase UI sometimes doesn't allow deletion due to foreign key constraints - Deleting from `profiles` first removes the constraint - Then deleting from `auth.users` removes the user completely --- ## 2. Automatic vs Manual Admin Assignment **Recommendation: MANUAL** ### Automatic (Email-based): - No setup needed - Less secure - Hard to manage multiple admins - Requires code changes ### Manual (Database): - More secure - Easy to manage - Can change anytime - Professional approach **Current Setup:** Manual (default `is_admin = FALSE`) **To make someone admin:** ```sql UPDATE profiles SET is_admin = TRUE WHERE email = 'user@example.com'; ``` **See `docs/ADMIN_MANAGEMENT.md` for full details.** --- ## 3.
