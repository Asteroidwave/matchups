# Admin Management Guide ## Automatic vs Manual Admin Assignment ### Option 1: Automatic (Based on Email) **Pros:** - No manual setup needed - Works immediately after signup - Consistent - same emails always get admin **Cons:** - Less secure (email can be spoofed) - Hard to manage multiple admins - Requires code changes to add/remove admins **How it works:** - When a user signs up, a trigger checks if their email matches a list - If match, automatically sets `is_admin = TRUE` ### Option 2: Manual (Recommended) **Pros:** - More secure - requires database access - Easy to manage - add/remove via SQL or admin panel - Can change admin status anytime - Better for production **Cons:** - Requires manual step after signup - Need to remember to do it **How it works:** - User signs up `is_admin = FALSE` by default - Admin goes to Supabase Updates `is_admin = TRUE` - Or use admin panel to promote/demote users ## Recommendation: MANUAL **Why:** More secure, flexible, and professional. You can always build an admin panel later to make it easier. ## Current Implementation Currently set to **MANUAL**: - New users sign up with `is_admin = FALSE` - You manually update in Supabase SQL Editor - Or use the admin panel (UserManagement component) ## Making Someone Admin Manually ### Method 1: SQL Editor (Supabase Dashboard) ```sql -- Make user admin by email UPDATE profiles SET is_admin = TRUE WHERE email = 'user@example.com'; -- Or by user ID UPDATE profiles SET is_admin = TRUE WHERE id = 'user-uuid-here'; ``` ### Method 2: Admin Panel (Future) - Go to `/admin` Users tab - Find the user - Click "Make Admin" button (if we add this feature) ## Removing Admin Status ```sql -- Remove admin status UPDATE profiles SET is_admin = FALSE WHERE email = 'user@example.com'; ``` ## Profile Refresh After Admin Change **Current Behavior:** - If you change admin status in Supabase - User needs to **refresh the page** or **logout/login** - The profile is cached in `AuthContext` **Why:** - Profile is loaded once on login - Stored in React state - Doesn't automatically refresh when database changes **Solution:** - User logs out logs back in - OR refresh the page (forces profile reload) - OR we can add a "Refresh Profile" button in admin panel ## Best Practice 1. **Change admin status** in Supabase 2. **Tell the user** to refresh their browser 3. **Or** they can logout/login We can add auto-refresh later, but for now, manual refresh is fine.
