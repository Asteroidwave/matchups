# Troubleshooting Guide ## Issue 1: Backend Not Running (ERR_CONNECTION_REFUSED) ### Symptoms - Console shows: `net::ERR_CONNECTION_REFUSED` for `localhost:3001` - Errors like: `TypeError: Failed to fetch` - Lobby shows "Loading..." forever - No data appears ### Solution **Start the backend server:** ```bash cd backend npm run dev ``` You should see: ``` Backend server running on port 3001 CORS enabled for: http://localhost:3000 ``` **Keep this terminal open** - the backend must stay running while you use the app. --- ## Issue 2: Admin Status Not Working ### Symptoms - Can't access `/admin` page - Profile shows `null` in debug page - Console shows: `infinite recursion detected in policy` ### Solution **Fix RLS policies in Supabase:** 1. Go to Supabase Dashboard SQL Editor 2. Run this SQL: ```sql -- Drop ALL existing policies DROP POLICY IF EXISTS "Users can view own profile" ON profiles; DROP POLICY IF EXISTS "Users can update own profile" ON profiles; DROP POLICY IF EXISTS "Users can insert own profile" ON profiles; DROP POLICY IF EXISTS "Admins can view all profiles" ON profiles; DROP POLICY IF EXISTS "Admins can update all profiles" ON profiles; -- Create function that bypasses RLS CREATE OR REPLACE FUNCTION is_user_admin(user_id UUID) RETURNS BOOLEAN AS $$ BEGIN RETURN EXISTS ( SELECT 1 FROM profiles WHERE id = user_id AND is_admin = TRUE ); END; $$ LANGUAGE plpgsql SECURITY DEFINER; -- Recreate policies (no circular references) CREATE POLICY "Users can view own profile" ON profiles FOR SELECT USING (auth.uid() = id); CREATE POLICY "Users can update own profile" ON profiles FOR UPDATE USING (auth.uid() = id) WITH CHECK (auth.uid() = id); CREATE POLICY "Users can insert own profile" ON profiles FOR INSERT WITH CHECK (auth.uid() = id); CREATE POLICY "Admins can view all profiles" ON profiles FOR SELECT USING ( auth.uid() = id OR is_user_admin(auth.uid()) ); CREATE POLICY "Admins can update all profiles" ON profiles FOR UPDATE USING ( auth.uid() = id OR is_user_admin(auth.uid()) ) WITH CHECK ( auth.uid() = id OR is_user_admin(auth.uid()) ); ``` 3. Refresh browser (hard refresh: Cmd+Shift+R) --- ## Issue 3: Profile Not Loading ### Check Profile in Supabase 1. Go to Supabase Dashboard Table Editor `profiles` 2. Find your row (email: `ritho@ralls.com`) 3. Verify `is_admin` is `TRUE` ### If Profile Doesn't Exist Run this SQL: ```sql INSERT INTO profiles (id, email, is_admin, bankroll, created_at, updated_at) VALUES ( '020ebbb7-c4ea-43cc-a5ce-d6954340b101', 'ritho@ralls.com', TRUE, 1000.00, NOW(), NOW() ) ON CONFLICT (id) DO UPDATE SET is_admin = TRUE, updated_at = NOW(); ``` --- ## Quick Checklist - [ ] Backend is running (`cd backend && npm run dev`) - [ ] Frontend is running (`npm run dev`) - [ ] RLS policies are fixed (run SQL above) - [ ] Profile exists in Supabase with `is_admin = TRUE` - [ ] Browser is refreshed (hard refresh) --- ## Still Not Working? 1. **Check browser console** for specific error messages 2. **Check backend terminal** for errors 3. **Verify environment variables** are set correctly 4. **Try logging out and back in** to refresh session
