# Testing Phase 3 Backend APIs ## Prerequisites 1. **Backend running**: `npm run dev` in `backend/` directory 2. **Admin user logged in**: Login as admin in frontend 3. **Get JWT token**: See methods below ## Getting Admin JWT Token ### Method 1: Browser Console (Easiest) 1. Open browser DevTools (F12) 2. Go to **Console** tab 3. Run: ```javascript const { supabase } = await import('@/lib/supabase/client'); const { data: { session } } = await supabase.auth.getSession(); console.log('Token:', session?.access_token); ``` 4. Copy the token ### Method 2: Network Tab 1. Open browser DevTools (F12) 2. Go to **Network** tab 3. Make any admin API call (e.g., create contest) 4. Click on the request 5. Look at **Request Headers** `Authorization: Bearer <token>` 6. Copy the token ### Method 3: localStorage 1. Open browser DevTools (F12) 2. Go to **Application** **Local Storage** `http://localhost:3000` 3. Look for `sb-<project-id>-auth-token` 4. Parse JSON and get `access_token` ## Running Tests ### Option 1: Automated Test Script 1. **Set token in backend/.env**: ```env TEST_ADMIN_TOKEN=your_jwt_token_here ``` 2. **Run test script**: ```bash cd backend npx tsx scripts/test-phase3-endpoints.ts ``` ### Option 2: Manual Testing with curl Replace `<TOKEN>` with your admin JWT token. #### Test 1: Fetch Track Data ```bash curl -X POST http://localhost:3001/api/admin/tracks/fetch \ -H "Authorization: Bearer <TOKEN>" \ -H "Content-Type: application/json" \ -d '{ "track": "GP", "date": "2025-11-02" }' ``` **Expected Response:** ```json { "success": true, "track_data_id": "uuid", "preview": { "races_count": 10, "horses_count": 102, "track_code": "GP", "date": "2025-11-02" } } ``` #### Test 2: Get Preview ```bash curl http://localhost:3001/api/admin/tracks/GP/2025-11-02/preview \ -H "Authorization: Bearer <TOKEN>" ``` #### Test 3: Create Contest with Track Data ```bash # First, get track_data_id from Test 1 response curl -X POST http://localhost:3001/api/admin/contests \ -H "Authorization: Bearer <TOKEN>" \ -H "Content-Type: application/json" \ -d '{ "track": "GP", "date": "2025-11-02", "track_data_id": "<track_data_id_from_test_1>", "contestType": "jockey_vs_jockey" }' ``` #### Test 4: Set Matchup Types ```bash # Use contest_id from Test 3 response curl -X PUT http://localhost:3001/api/admin/contests/<contest_id>/matchup-types \ -H "Authorization: Bearer <TOKEN>" \ -H "Content-Type: application/json" \ -d '{ "matchup_types": ["jockey_vs_jockey", "trainer_vs_trainer"] }' ``` #### Test 5: Get Contest Status ```bash curl http://localhost:3001/api/admin/contests/<contest_id>/status \ -H "Authorization: Bearer <TOKEN>" ``` #### Test 6: Create Operation ```bash curl -X POST http://localhost:3001/api/admin/operations \ -H "Authorization: Bearer <TOKEN>" \ -H "Content-Type: application/json" \ -d '{ "operation_type": "fetch_track_data", "input_data": { "track": "GP", "date": "2025-11-02" } }' ``` #### Test 7: Get Operation Status ```bash # Use operation_id from Test 6 response curl http://localhost:3001/api/admin/operations/<operation_id> \ -H "Authorization: Bearer <TOKEN>" ``` ## Expected Results ### Success Indicators: - Status code: `200` or `201` - Response includes `success: true` - Data is returned correctly - No errors in backend console ### Common Issues: **401 Unauthorized:** - Token expired or invalid - Get fresh token from browser **403 Forbidden:** - User is not admin - Check `is_admin = true` in Supabase **404 Not Found:** - Track/date combination doesn't exist in MongoDB - Try different track/date **500 Internal Server Error:** - Check backend console for details - Verify MongoDB connection - Verify Supabase connection ## Verification Checklist After running tests, verify in Supabase: 1. **Track Data Created**: ```sql SELECT * FROM track_data WHERE track_code = 'GP' AND date = '2025-11-02'; ``` 2. **Contest Created**: ```sql SELECT id, track, date, status, track_data_id FROM contests WHERE track = 'GP' AND date = '2025-11-02'; ``` 3. **Matchup Types Set**: ```sql SELECT matchup_types, status FROM contests WHERE id = '<contest_id>'; ``` 4. **Operation Created**: ```sql SELECT * FROM operations ORDER BY created_at DESC LIMIT 1; ``` ## Next Steps After successful tests: 1.
