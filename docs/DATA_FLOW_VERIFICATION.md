# Data Flow Verification ## Current Data Flow: MongoDB Redis Frontend ### Complete Flow 1. **Lobby Page** (`/lobby`) - Fetches contests from `/api/contests` (hardcoded for Nov 2, 2025) - Verifies each track has data by calling `/api/tracks/:track/entries` - Shows only contests with available data - User clicks "Enter" Stores `selectedTrack` and `selectedDate` in `sessionStorage` 2. **Backend** (`/api/tracks/:track/entries`) - **First Request**: Checks Redis cache Miss Queries MongoDB Stores in Redis Returns data - **Subsequent Requests**: Checks Redis cache Hit Returns cached data (no MongoDB call) - **Cache TTL**: 24 hours for past data (Nov 2, 2025) 3. **Matchups Page** (`/matchups`) - `AppContext.loadData()` reads `selectedTrack` and `selectedDate` from `sessionStorage` - Calls `loadAndMergeAllTracks(date, track)` with selected track - `loadTrackData()` `getTrackEntries()` Fetches from backend API - Backend returns cached data from Redis (fast!) - Data flows: Backend Frontend API `AppContext` `StartersWindow` 4. **Starters Panel** - Receives connections data from `AppContext` - Displays horses with program numbers - Sorted by program number (1, 2, 3...) - Shows only selected track (if one was chosen) ### All 6 Tracks Configured - **BAQ** (Belmont Park) - **CD** (Churchill Downs) - **DMR** (Del Mar) - **GP** (Gulfstream Park) - **LRL** (Laurel Park) - **MNR** (Mountaineer Park) All configured for November 2, 2025. ### Redis Caching Active -
