# Why Couldn't the Script Get a Session Token? ## The Problem The `clear-cache.ts` script uses the **service role key** to connect to Supabase: ```typescript const supabase = createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, { auth: { autoRefreshToken: false, persistSession: false, }, }); ``` ## Why Service Role Key Can't Get Sessions 1. **Service Role Key ≠ User Authentication** - Service role keys are for **server-side operations** - They bypass Row Level Security (RLS) - They don't have user sessions because there's no authenticated user 2. **Sessions Require User Login** - Sessions are created when a **user logs in** (email/password, OAuth, etc.) - The service role key is not a user - it's an admin key - `getSession()` will always return `null` with a service role key 3. **Different Use Cases** - **Service Role Key**: Direct database access, admin operations, server-side scripts - **User Session Token**: User-specific operations, API calls on behalf of a logged-in user ## Solutions ### Option 1: Direct Redis Script (Recommended) I've created a new script `clear-redis.ts` that connects directly to Upstash Redis: ```bash npx tsx scripts/clear-redis.ts ``` This script: - Uses Upstash credentials from `backend/.env` - Doesn't need user sessions - Clears Redis cache directly ### Option 2: Manual via Upstash Dashboard 1. Go to [Upstash Console](https://console.upstash.com/) 2. Select your Redis database 3. Use the CLI to delete keys: ```redis DEL contests:active ``` ### Option 3: Backend API (When Logged In) If you're logged in as admin in your browser, you can call the backend API: ```bash # Get your token from browser localStorage or use browser console curl -X POST http://localhost:3001/api/admin/contests/cache/clear \ -H "Authorization: Bearer YOUR_ADMIN_TOKEN" ``` ## Summary - **Service role key** = Admin database access (no user session) - **User session token** = Authenticated user operations (requires login) - **Solution**: Use `clear-redis.ts` script or clear manually via Upstash dashboard
