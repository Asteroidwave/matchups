# Debug: Why Contests Table is Empty ## The Problem - Contests table in Supabase is empty - Upstash Redis has no cached contests (because there are none to cache) - Lobby shows "No Contests Available" ## Why This Happens **Contests are NOT automatically created** - they must be created manually by an admin. ## Solutions ### Option 1: Create Contest via Admin Panel (Recommended) 1. **Go to Admin Panel:** - Login as admin - Go to `/admin` - Click "Contests" tab 2. **Fill out the form:** - Track: Select one (e.g., "Gulfstream Park (GP)") - Date: Enter a date (e.g., `2025-11-02`) - Contest Type: "Jockey vs Jockey" (default) - Entry Fee: Leave empty (optional) - Prize Pool: Leave empty (optional) 3. **Click "Create Contest"** 4. **Check for errors:** - If you see an error, check: - Browser console (F12) - Backend terminal logs - Network tab (check `/api/admin/contests` request) ### Option 2: Create Contest via SQL (Quick Test) 1. **Go to Supabase Dashboard SQL Editor** 2. **Run this:** ```sql INSERT INTO contests (track, date, contest_type, is_active) VALUES ('GP', '2025-11-02', 'jockey_vs_jockey', TRUE) RETURNING *; ``` 3. **Verify:** ```sql SELECT * FROM contests; ``` ### Option 3: Use Seed Script (If You Have One) ```bash cd backend npm run seed:contests # or tsx scripts/seed-contests.ts ``` ## Debugging Contest Creation ### If Admin Panel Creation Fails: 1. **Check Browser Console:** - Open DevTools (F12) - Look for errors in Console tab - Check Network tab for `/api/admin/contests` request 2. **Check Backend Logs:** - Look for: - ` Contest created in Supabase` - ` Error creating contest:` - Any Supabase errors 3. **Common Issues:** **Issue: "Supabase not configured"** - Fix: Check `backend/.env` has `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` - Restart backend **Issue: "RLS policy violation"** - Fix: Check RLS policies in `docs/FRESH_SUPABASE_SETUP.sql` - Make sure "Admins can create contests" policy exists **Issue: "Unauthorized" or "401"** - Fix: Make sure you're logged in as admin - Check your profile has `is_admin = TRUE` in Supabase **Issue: "Duplicate key" or "UNIQUE constraint"** - Fix: A contest with same track/date/type already exists - Either delete it or use different track/date ## Verify Contest Was Created 1. **In Supabase:** - Table Editor `contests` - Should see your contest 2. **Via API:** ```bash curl http://localhost:3001/api/contests ``` Should return your contest 3. **In Lobby:** - Refresh `/lobby` - Contest should appear ## Quick Test Run this to verify everything works: ```bash # 1. Check it was created curl http://localhost:3001/api/contests ``` ## Next Steps Once you create a contest: 1. It should appear in admin panel "Manage Contests" 2.
