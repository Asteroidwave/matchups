# How to Clear Cache ## Quick Start Simply run this command in your terminal: ```bash npx tsx scripts/clear-cache.ts ``` ## What This Does This script will: 1. Clear all **contests** from Supabase 2. Clear all **track_data** from Supabase 3. Clear all **matchups** from Supabase 4. Clear all **operations** from Supabase ## Requirements ### 1. Environment Variables Make sure your `.env.local` file has: ```env NEXT_PUBLIC_SUPABASE_URL=your_supabase_url SUPABASE_SERVICE_ROLE_KEY=your_service_role_key ``` **Note**: The script uses the **service role key** (not the anon key) because it needs admin privileges to delete data. ### 2. Install tsx (if needed) The `npx` command will automatically download `tsx` if it's not installed, but if you want to install it globally: ```bash npm install -g tsx ``` Then you can run: ```bash tsx scripts/clear-cache.ts ``` ## Step-by-Step Instructions ### 1. Open Terminal Navigate to your project directory: ```bash cd /Users/briangitonga/Downloads/project ``` ### 2. Run the Script ```bash npx tsx scripts/clear-cache.ts ``` ### 3. Expected Output You should see: ``` Clearing all cached data... Clearing contests table... Cleared X contests Clearing track_data table... Cleared X track_data entries Clearing matchups table... Cleared X matchups Clearing operations table... Cleared X operations Cache clearing complete! Note: Redis cache will expire naturally or can be cleared via backend API To clear Redis cache, call: POST /api/admin/contests/cache/clear Ready to test fresh flow: 1. Fetch track data from MongoDB 2. Preview track stats 3. Create contest 4. Set matchup types with per-type settings 5. Calculate matchups 6. Enable contest visibility 7. Contest appears in lobby ``` ## Troubleshooting ### Error: Missing NEXT_PUBLIC_SUPABASE_URL **Solution**: Make sure `.env.local` exists and has the correct Supabase URL. ### Error: Missing SUPABASE_SERVICE_ROLE_KEY **Solution**: 1. Go to your Supabase project dashboard 2. Navigate to Settings API 3. Copy the "service_role" key (not the "anon" key) 4. Add it to `.env.local` as `SUPABASE_SERVICE_ROLE_KEY` ### Error: Cannot find module 'dotenv' **Solution**: Install dependencies: ```bash npm install ``` ### Error: Permission denied **Solution**: The service role key might not have the right permissions. Make sure you're using the service_role key, not the anon key. ## Alternative: Clear via Backend API If the script doesn't work, you can also clear the backend cache via API: ```bash curl -X POST http://localhost:3001/api/admin/contests/cache/clear \ -H "Authorization: Bearer YOUR_ADMIN_TOKEN" ``` But this only clears Redis cache, not Supabase data. ## What Gets Cleared - **Contests**: All contest records - **Track Data**: All fetched track data - **Matchups**: All calculated matchups - **Operations**: All operation tracking records ## What Doesn't Get Cleared - **Profiles**: User profiles remain - **Tracks**: Track visibility settings remain - **Redis Cache**: Must be cleared separately via backend API ## After Clearing Once you've cleared the cache, you can test the full flow: 1. **Fetch Track Data**: Go to admin Contests Create New Contest 2. **Preview**: See track stats (races, horses, jockeys, trainers, sires) 3. **Create Contest**: Fill in contest details and create 4. **Set Matchup Types**: Select types and configure per-type settings 5. **Calculate**: Wait for matchups to be calculated 6. **Enable Visibility**: Toggle contest to visible 7. **View in Lobby**: Contest appears in lobby for users
