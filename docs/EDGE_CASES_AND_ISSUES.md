# Edge Cases and Potential Issues Found ## Code Review Findings ### 1. **Authentication & Session Issues** #### Issue: Session Retrieval Inconsistency FIXED - **Location**: `MatchupTypesDialog.tsx`, `CreateContestForm.tsx` - **Problem**: Using `{ session }` instead of `{ data: { session } }` - **Impact**: "You must be logged in" errors even when logged in - **Status**: Fixed #### Issue: Profile Creation Race Condition - **Location**: `AuthContext.tsx` - `loadProfile()` - **Problem**: If profile trigger fails, user might not have profile created - **Impact**: User exists in auth but not in profiles table - **Mitigation**: Code handles PGRST116 error and creates profile #### Issue: Session Expiry Handling - **Location**: Multiple components - **Problem**: No automatic session refresh when token expires - **Impact**: Users get logged out unexpectedly - **Recommendation**: Use `useSessionRefresh` hook more broadly ### 2. **Contest Creation Flow** #### Issue: Track Data Fetch Failure - **Location**: `CreateContestForm.tsx` - **Problem**: If MongoDB fetch fails, user sees error but no retry option - **Impact**: Poor UX when network is flaky - **Recommendation**: Add retry button #### Issue: Duplicate Contest Creation - **Location**: `CreateContestForm.tsx` - **Problem**: No check for existing contest with same track/date - **Impact**: Can create duplicate contests - **Recommendation**: Check before creating #### Issue: Track Data Preview Not Cached - **Location**: `CreateContestForm.tsx` - **Problem**: If user closes preview, they need to re-fetch - **Impact**: Unnecessary API calls - **Recommendation**: Cache preview in component state ### 3. **Matchup Calculation** #### Issue: Calculation Timeout - **Location**: `backend/src/services/matchupCalculation.ts` - **Problem**: No timeout for long-running calculations - **Impact**: Can hang if data is very large - **Recommendation**: Add timeout (e.g., 5 minutes) #### Issue: Incomplete Matchup Types - **Location**: `MatchupTypesDialog.tsx` - **Problem**: "Mixed" matchup type not implemented in backend - **Impact**: Selecting "Mixed" will fail - **Recommendation**: Implement or disable "Mixed" option #### Issue: No Progress Updates - **Location**: `backend/src/services/matchupCalculation.ts` - **Problem**: No progress updates during calculation - **Impact**: User doesn't know if it's stuck - **Recommendation**: Update operation progress periodically ### 4. **Data Loading & Caching** #### Issue: Session Storage Race Condition - **Location**: `AppContext.tsx` - **Problem**: Uses `sessionStorage` but doesn't validate data exists - **Impact**: Can try to load track that doesn't exist - **Recommendation**: Validate track exists before loading #### Issue: Cache Invalidation - **Location**: `backend/src/routes/contests.ts` - **Problem**: Cache TTL is 5 minutes, but admin changes might not be reflected - **Impact**: Stale data shown to users - **Recommendation**: Better cache invalidation strategy #### Issue: Multiple Track Loading - **Location**: `lib/ingest.ts` - **Problem**: No limit on concurrent track loads - **Impact**: Memory issues with many tracks - **Recommendation**: Add concurrency limit ### 5. **User Experience** #### Issue: No Loading States - **Location**: Multiple components - **Problem**: Some operations don't show loading indicators - **Impact**: Users don't know if action is processing - **Recommendation**: Add loading states everywhere #### Issue: Error Messages Not Clear - **Location**: Multiple components - **Problem**: Generic error messages like "Failed to..." - **Impact**: Users don't know what went wrong - **Recommendation**: More specific error messages #### Issue: No Confirmation Dialogs - **Location**: `ContestManagement.tsx` - **Problem**: Delete contest has confirmation, but toggle visibility doesn't - **Impact**: Accidental visibility changes - **Recommendation**: Add confirmation for important actions ### 6. **Edge Cases** #### Edge Case: Empty Track Data - **Location**: `backend/src/services/matchupCalculation.ts` - **Problem**: What if track data has no entries? - **Impact**: Calculation fails silently - **Recommendation**: Return clear error message #### Edge Case: Very Small Track Data - **Location**: `backend/src/services/matchupCalculation.ts` - **Problem**: If only 1-2 connections, can't generate matchups - **Impact**: Returns empty matchups with no explanation - **Recommendation**: Check and warn if not enough connections #### Edge Case: Expired Track Data - **Location**: `backend/src/routes/admin-track-data.ts` - **Problem**: No check if track data is for past date - **Impact**: Can create contests for past dates - **Recommendation**: Validate date is not too far in past #### Edge Case: Concurrent Requests - **Location**: Multiple components - **Problem**: No debouncing on rapid clicks - **Impact**: Multiple duplicate requests - **Recommendation**: Add request debouncing #### Edge Case: Network Interruption - **Location**: All API calls - **Problem**: No retry logic for failed requests - **Impact**: One network hiccup breaks flow - **Recommendation**: Add retry with exponential backoff ### 7. **Security Concerns** #### Issue: Admin Access Check - **Location**: `components/auth/ProtectedRoute.tsx` - **Problem**: Relies on cached profile for immediate access - **Impact**: Could be bypassed if cache is manipulated - **Recommendation**: Always verify with backend on critical actions #### Issue: Session Validation - **Location**: Backend middleware - **Problem**: Only checks if user is admin, not if session is valid - **Impact**: Expired sessions might still work - **Recommendation**: Validate session expiry ### 8. **Database Issues** #### Issue: Orphaned Records - **Location**: Multiple tables - **Problem**: If contest deleted, track_data might remain - **Impact**: Storage bloat - **Recommendation**: Add cascade deletes or cleanup job #### Issue: Race Conditions - **Location**: `operations` table - **Problem**: Multiple operations can be created simultaneously - **Impact**: Confusion about which operation is active - **Recommendation**: Add operation locking ## Testing Checklist ### Authentication Tests -
