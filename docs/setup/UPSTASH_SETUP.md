# Upstash Redis Setup Guide ## Step-by-Step Instructions ### 1. Create Upstash Account 1. Go to [https://upstash.com/](https://upstash.com/) 2. Click **"Sign Up"** (or **"Log In"** if you already have an account) 3. Sign up with: - GitHub (recommended - easy integration) - Email - Google ### 2. Create Redis Database 1. Once logged in, you'll see the **Upstash Console** 2. Click **"Create Database"** or **"New Database"** 3. Fill in the form: - **Name**: `matchups-cache` (or any name you prefer) - **Type**: Select **"Regional"** (free tier) - **Region**: Choose closest to you (e.g., `us-east-1`, `eu-west-1`) - **Primary Region**: Same as region above - **Rest API**: **Enabled** (this is what we need!) - **Eviction**: `No eviction` (for free tier) 4. Click **"Create"** ### 3. Get Your Credentials 1. After creating the database, you'll see the database details page 2. Scroll down to **"REST API"** section 3. You'll see: - **UPSTASH_REDIS_REST_URL**: `https://xxxxx.upstash.io` - **UPSTASH_REDIS_REST_TOKEN**: `xxxxx...` (long token) 4. **Copy both values** - you'll need them in the next step ### 4. Add to Backend Environment 1. Open `backend/.env` (create it if it doesn't exist) 2. Add these lines: ```env UPSTASH_REDIS_REST_URL=https://your-redis-instance.upstash.io UPSTASH_REDIS_REST_TOKEN=your_redis_token_here ``` **Important**: Replace `your-redis-instance.upstash.io` and `your_redis_token_here` with the actual values from Step 3. ### 5. Verify Setup 1. **Restart your backend server**: ```bash cd backend npm run dev ``` 2. **Check the console output** - you should see: ``` Redis cache initialized (Upstash) ``` 3. **If you see this instead**: ``` ℹ️ Redis not configured, using in-memory cache ``` Then check: - Did you add the env variables correctly? - Did you restart the server? - Are the credentials correct? ### 6. Test the Cache 1. Make a request to your backend: ```bash curl http://localhost:3001/api/tracks/GP/entries?date=2025-11-02 ``` 2. Check the backend logs - first request should show MongoDB query 3. Make the same request again: ```bash curl http://localhost:3001/api/tracks/GP/entries?date=2025-11-02 ``` 4. Check logs - second request should be much faster (cached, no MongoDB call) ## Free Tier Limits Upstash free tier includes: - **10,000 commands/day** - More than enough for development - **256 MB storage** - Plenty for caching contest data - **Global replication** - Fast access worldwide ## Troubleshooting ### Issue: "Redis not configured" message **Solution**: 1. Check `backend/.env` file exists and has the correct variables 2. Make sure variable names are exactly: - `UPSTASH_REDIS_REST_URL` - `UPSTASH_REDIS_REST_TOKEN` 3. Restart the backend server ### Issue: Connection errors **Solution**: 1. Verify credentials are correct (no extra spaces) 2. Check if your Upstash database is active (not paused) 3. Make sure REST API is enabled in Upstash dashboard ### Issue: Still seeing MongoDB calls every time **Solution**: 1. Check backend logs for cache hits/misses 2. Verify Redis is actually being used (should see " Redis cache initialized") 3. Check cache TTL - past data should cache for 24 hours ## Next Steps Once Redis is set up: - All MongoDB queries will be cached - No repeated calls during testing - Faster response times - Ready for production scaling ## Monitoring (Optional) You can monitor cache usage in Upstash dashboard: 1. Go to your database in Upstash console 2. Click **"Metrics"** tab 3. See cache hits, commands, storage usage
