# Connection Generation Logic ## Key Principle **Entries and Results are separate data sources:** - **Entries**: All horses that were entered (including scratched and AE) - **Results**: Only horses that actually raced (finished the race) ## How Connections Are Generated ### 1. Apps (Appearances) - **Counted from ENTRIES** (all non-scratched horses, including AE) - This determines matchup eligibility (`min_appearances` threshold) - Why: A jockey/trainer/sire connection exists even if the horse was AE or scratched ### 2. Salary - **Counted from ENTRIES** (all non-scratched horses, including AE) - Used for matchup salary matching - Why: Salary is based on morning line odds, which exist in entries ### 3. Points - **Counted from RESULTS** (only horses that actually raced and finished) - Only used for results page, NOT for matchup eligibility - Why: Points come from race results (win/place/show payoffs) ## Example **Entries Data:** - Horse A: Jockey "Smith", scratched = false, AE = false - Horse B: Jockey "Smith", scratched = false, AE = true - Horse C: Jockey "Smith", scratched = true **Results Data:** - Horse A: Finished 1st, points = 50 **Connection for Jockey "Smith":** - Apps: 2 (Horse A + Horse B, excluding scratched Horse C) - Salary: Sum of salaries for Horse A + Horse B - Points: 50 (only from Horse A that actually raced) ## Why This Matters 1. **Matchup Eligibility**: Based on apps from entries (not results) - If a jockey has 5 entries (including AE), they have 5 apps - Even if only 3 actually raced, they still have 5 apps for matchup purposes 2. **Results Page**: Shows points from results - Only horses that actually raced have points - This is separate from matchup calculations 3. **Data Mismatch**: Entries and results won't always match - Some horses scratch and won't be in results - Some horses are AE and won't be in results - This is expected and handled correctly
