# Quick Test Guide for Phase 3 Endpoints ## Step 1: Get Admin Token ### Option A: Browser Console (Easiest) 1. **Open your app in browser** (logged in as admin) 2. **Open DevTools** (F12) 3. **Go to Console tab** 4. **Run this code**: ```javascript const { supabase } = await import('/lib/supabase/client.js'); const { data: { session } } = await supabase.auth.getSession(); console.log('Token:', session?.access_token); ``` 5. **Copy the token** that appears ### Option B: Network Tab 1. **Open DevTools** (F12) 2. **Go to Network tab** 3. **Make any admin API call** (e.g., go to admin page) 4. **Click on a request** with `/api/admin/` in the URL 5. **Look at Request Headers** `Authorization: Bearer <token>` 6. **Copy the token** ### Option C: localStorage 1. **Open DevTools** (F12) 2. **Go to Application** **Local Storage** `http://localhost:3000` 3. **Find key** like `sb-<project-id>-auth-token` 4. **Copy the JSON**, parse it, get `access_token` ## Step 2: Test Endpoints ### Quick Test with curl Replace `<TOKEN>` with your admin token. ```bash # Test 1: Health check (no auth needed) curl http://localhost:3001/health # Test 2: Fetch Track Data (should work if backend is running) curl -X POST http://localhost:3001/api/admin/tracks/fetch \ -H "Authorization: Bearer <TOKEN>" \ -H "Content-Type: application/json" \ -d '{ "track": "GP", "date": "2025-11-02" }' # Test 3: Get Preview curl http://localhost:3001/api/admin/tracks/GP/2025-11-02/preview \ -H "Authorization: Bearer <TOKEN>" ``` ### Automated Test Script 1. **Add token to backend/.env**: ```env TEST_ADMIN_TOKEN=your_token_here ``` 2. **Run test script**: ```bash cd backend npx tsx scripts/test-phase3-endpoints.ts ``` ## Step 3: Expected Results ### Success Indicators: - Status code: `200` or `201` - Response includes `success: true` - Data returned correctly - No errors in backend console ### Common Issues: **401 Unauthorized:** - Token expired Get fresh token - Token invalid Check you copied it correctly **403 Forbidden:** - User not admin Check `is_admin = true` in Supabase **404 Not Found:** - Track/date doesn't exist in MongoDB Try different track/date **500 Internal Server Error:** - Check backend console for details - Verify MongoDB connection - Verify Supabase connection ## Step 4: Verify in Supabase After successful tests, check the database: ```sql -- Check track_data was created SELECT * FROM track_data WHERE track_code = 'GP' AND date = '2025-11-02'; -- Check contest was created SELECT id, track, date, status, track_data_id FROM contests WHERE track = 'GP' AND date = '2025-11-02'; -- Check matchups after calculation SELECT matchup_type, COUNT(*) as count FROM matchups WHERE contest_id = '<contest_id>' GROUP BY matchup_type; ``` ## Next Steps After successful tests: 1. Endpoints verified 2. Continue with Phase 4 (Frontend Components)
