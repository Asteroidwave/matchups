Create Supabase project 2. Run SQL scripts to create tables: - `profiles` table - `contests` table - Auto-create profile trigger 3. Get credentials (URL, anon key, service role key) 4. Add to environment variables ### 2. **Environment Variables** **Frontend** (`.env.local`): ```env NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ... NEXT_PUBLIC_BACKEND_URL=http://localhost:3001 ``` **Backend** (`backend/.env`): ```env SUPABASE_URL=https://xxxxx.supabase.co SUPABASE_SERVICE_ROLE_KEY=eyJ... # ... other existing vars ``` ### 3. **Make Yourself Admin** After creating your account, run this SQL in Supabase: ```sql UPDATE profiles SET is_admin = TRUE WHERE email = 'your-email@example.com'; ``` ### 4. **Test Everything** 1. **Sign up**: Go to `/signup` and create account 2. **Login**: Go to `/login` and sign in 3. **Admin access**: Make yourself admin, then go to `/admin` 4. **Create contest**: Use admin panel to create a contest 5. **User management**: Adjust a user's bankroll --- ## What Still Needs Work ### 1. **Bankroll Integration** (Pending) - Currently bankroll is still in localStorage - Need to: - Load bankroll from Supabase on login - Update bankroll in Supabase when it changes - Sync AppContext with Supabase profile ### 2. **Backend Contest Integration** (Pending) - Currently contests are hardcoded - Need to: - Update `backend/src/utils/contests.ts` to read from Supabase - Update `backend/src/routes/contests.ts` to query Supabase - Remove hardcoded contest list ### 3. **Entry System** (Future) - Store user entries in Supabase - Entry validation - Entry history page --- ## Files Created ### Frontend - `lib/supabase/client.ts` - Supabase client - `contexts/AuthContext.tsx` - Auth state management - `app/login/page.tsx` - Login page - `app/signup/page.tsx` - Signup page - `app/admin/page.tsx` - Admin dashboard - `components/auth/ProtectedRoute.tsx` - Route protection - `components/admin/CreateContestForm.tsx` - Contest creation - `components/admin/UserManagement.tsx` - User management ### Backend - `backend/src/routes/admin.ts` - Admin API routes - `backend/src/utils/supabase.ts` - Backend Supabase client ### Documentation - `docs/SUPABASE_SETUP.md` - Setup guide - `docs/AUTH_AND_ADMIN_IMPLEMENTATION.md` - This file --- ## How to Use ### For Users: 1.
