# Salary and AVPA Implementation ## Overview The matchup calculation now uses: 1. **Talha's Salary Calculation Method** - Lowest odds per race for jockeys 2. **AVPA (30/90 Days)** - From MongoDB performance collections 3. **Past Race Data** - Last 5 races for each connection ## Salary Calculation ### Talha's Method (from unified_matchup_generator.py) **For Jockeys:** - Use **lowest odds per race** (not all odds) - If a jockey has multiple horses in a race, only count the lowest odds - Then calculate salary from those odds **For Trainers/Sires:** - Use **all odds** (not per-race) - Sum salaries from all horses **Salary Bins (Regular):** - 0.01-0.79: 2600 - 0.8-0.99: 2500 - 1.0-1.19: 2400 - 1.2-1.39: 2300 - 1.4-1.59: 2200 - 1.6-1.79: 2100 - 1.8-1.99: 2000 - 2.0-2.49: 1900 - 2.5-2.99: 1700 - 3.0-3.49: 1600 - 3.5-3.99: 1500 - 4.0-4.49: 1400 - 4.5-5.99: 1300 - 6.0-7.99: 1200 - 8.0-9.99: 1000 - 10.0-11.99: 900 - 12.0-14.99: 800 - 15.0-19.99: 700 - 20.0-29.99: 600 - 30.0-49.99: 400 - 50.0+: 200 **Also Eligible:** - 0-5.99: 300 - 6.0-12.99: 200 - 13.0+: 100 ## AVPA (Average Value Per Appearance) **Source**: MongoDB performance collections - `jockeyperformances` - `jockey_key` `thirty_days.average_value_per_appearance`, `ninety_days.average_value_per_appearance` - `trainerperformances` - `trainer_key` same structure - `sireperformances` - `key` same structure **Keys extracted from entries:** - Jockey: `jockey.key` - Trainer: `trainer.key` - Sire: `sire.referenceNumber` or `sire.key` ## Past Race Data **Last 5 races** for each connection: - Queries historical results from MongoDB - Shows track, date, race, horse, place, points, salary - Helps with matchup generation by showing recent performance ## Implementation 1. **Extract keys** from entries during transformation (`trackDataTransform.ts`) 2. **Load performance data** once at startup (cached) 3. **Calculate salaries** using Talha's method (lowest odds per race for jockeys) 4. **Fetch AVPA** from performance collections using keys 5. **Fetch past races** from historical results (last 5) ## Data Flow ``` Entries Extract keys Generate connections Calculate salary (Talha's method) Load AVPA from MongoDB Fetch past races (last 5) Generate matchups ```
