# Decisions: Tracks Table & Inactivity ## 1. Tracks Table - Should We Keep It? ### Current State The `tracks` table has: - GP, LRL, DMR, BAQ, CD, MNR (test data) - `is_active` flag for visibility - Used by `TrackManagement` component ### Purpose The tracks table is used for **track visibility management**: - Admins can toggle which tracks are visible globally - This is **separate** from contest data - Tracks are fetched fresh from MongoDB for each contest ### Recommendation **Option A: Keep Tracks Table (Recommended)** - **Why**: Allows admins to control which tracks players can see, even if contests exist - **How**: Clear test data, make it dynamic (tracks added when contests created) - **Use case**: Hide a track while keeping its contests (maintenance mode) **Option B: Remove Tracks Table** - **Why**: Simpler - tracks determined by active contests only - **How**: Delete table, remove TrackManagement component, remove tracks tab - **Use case**: If you don't need global track visibility control ### My Recommendation **Keep the tracks table** but: 1. Clear the test data 2. Make it dynamic - tracks added when: - Admin creates first contest for a track - Or admin manually adds via TrackManagement 3. Use it for visibility management only **Script to clear tracks**: ```bash npx tsx scripts/clear-tracks-table.ts ``` --- ## 2. Session Refresh Hook - Removed **Done**: Removed `useSessionRefresh` from admin page. **Now**: Session refresh is handled by: - Auth interceptor (automatic refresh before API calls) - Keep-alive mechanism (every 5 minutes) --- ## 3. Inactivity Issue - Root Cause & Solution ### Why Buttons Become Inactive **Primary Cause**: **Session Token Expiry** - Supabase JWT tokens expire after ~1 hour - After expiry, API calls fail with 401 - Buttons click but nothing happens (no error shown) **Secondary Causes**: 1. **Event handlers not firing** - React state issues 2. **Network connection closed** - Browser closes idle connections 3. **Auth interceptor not working** - Not all API calls use it ### Solution Implemented **1. Auth Interceptor** (already done) - Automatically refreshes token before API calls - Retries on 401 errors - All admin API calls use `fetchWithAuth` **2. Keep-Alive Mechanism** (just added) - Refreshes session every 5 minutes - Prevents token from expiring - Keeps connection alive **3. Better Error Handling** (recommended) - Show errors when API calls fail - Allow user to retry - Clear error messages ### How to Test 1. Stay on admin page for 10+ minutes 2. Try clicking buttons 3. Check browser console for: - Session refresh logs - 401 errors - Network errors ### If Still Not Working **Check**: 1. Are all API calls using `fetchWithAuth`? (Yes) 2. Is keep-alive running? (Yes, every 5 min) 3. Are there console errors? (Check browser console) **Additional Fixes**: - Add error boundaries - Show loading states - Add retry buttons - Log all API failures --- ## Summary 1. **Tracks Table**: Keep it, clear test data, make it dynamic 2. **Session Refresh**: Removed hook, using interceptor + keep-alive 3. **Inactivity**: Fixed with keep-alive (every 5 min) + auth interceptor --- ## Next Steps 1. **Clear tracks table** (optional): ```bash npx tsx scripts/clear-tracks-table.ts ``` 2. **Test inactivity**: - Stay on admin page for 10+ minutes - Try clicking buttons - Report if still not working 3. **Monitor**: - Check console for keep-alive logs - Check for 401 errors - Report any issues
