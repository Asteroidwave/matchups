# Matchups Backend API Backend API server for the Matchups platform. ## Setup ### 1. Install Dependencies ```bash npm install ``` ### 2. Configure Environment Variables Copy `.env.example` to `.env` and fill in your credentials: ```bash cp .env.example .env ``` Edit `.env` with your: - MongoDB connection strings (Talha's credentials) - Supabase URL and keys - Upstash Redis credentials (optional) ### 3. Run Development Server ```bash npm run dev ``` Server will start on `http://localhost:3001` ### 4. Build for Production ```bash npm run build npm start ``` ## API Endpoints ### Health Check - `GET /health` - Check if server is running ### Tracks - `GET /api/tracks/:track/entries?date=YYYY-MM-DD` - Get race entries - `GET /api/tracks/:track/results?date=YYYY-MM-DD` - Get race results ### Connections - `GET /api/connections?type=jockey|trainer|sire&track=GP&date=YYYY-MM-DD` - Get connections ### Matchups - `POST /api/matchups/generate` - Generate matchups (to be implemented) - `GET /api/matchups?contestId=xxx` - Get matchups (to be implemented) ## Deployment to Railway 1. Create account at https://railway.app 2. Connect your GitHub repository 3. Add environment variables in Railway dashboard 4. Deploy! Railway will automatically detect this is a Node.js project and build it. ## Environment Variables - `MONGODB_URI_COMMON` - MongoDB connection for common-central database - `MONGODB_URI_STAGING` - MongoDB connection for staging-gos database - `SUPABASE_URL` - Supabase project URL - `SUPABASE_ANON_KEY` - Supabase anonymous key - `SUPABASE_SERVICE_ROLE_KEY` - Supabase service role key - `UPSTASH_REDIS_REST_URL` - Upstash Redis URL (optional) - `UPSTASH_REDIS_REST_TOKEN` - Upstash Redis token (optional) - `PORT` - Server port (default: 3001) - `CORS_ORIGIN` - Frontend URL for CORS (default: http://localhost:3000)
